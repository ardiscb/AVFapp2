Data Research, Project Intent

Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

Since payments are more secure when logged into accounts such as Twitter, using the oAuth to validate a person’s identity is a great perk when dealing with payments within an app.

Visit Programmableweb.com and search for JSON‐P data sources. Why would we need JSON‐P, specifically?

We need JSON‐P to talk to the APIs, such as Twitter. An AJAX call pulls in the Twitter feed to the app. JSON-P is used specifically for cross-platform applications.

Based on the JSON‐P search results: what data resource(s) do you think might pair well with which types of Native device features?

I think a good mash-up would be with the Contacts native feature and the GitHub Jobs API. It would be great to allow a user to add a GitHub job contact from the GitHub job list page.

Please describe any design and functionality intentions you may have for your Demo app components this month.

I have designed the demo app to be somewhat of an app for organizing different tasks or functions. I want to implement Media, so the user can record audio files, such as quick to-do voice reminders. I also plan to implement Geolocation, Contacts, and Camera native features.

Briefly describe some past 'lessons learned' regarding workflow as it relates to research, design, development, etc.

I've learned to get the functionality of things working first and then work on the detailed styles or extra features later. I’ve also started drawing out on paper a rough sketch of how I want each application laid out.
